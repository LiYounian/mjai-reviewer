# 牌谱 JSON 格式说明

> 本项目涉及两种牌谱 JSON：**雀魂原始格式**（`data/raw/`）和**天凤格式**（`data/games/`）。
> 前者是雀魂 protobuf 解码后的直接产物，后者是由前者转换而来的通用格式。
> 日常使用（统计、AI 复盘）只需关心天凤格式；雀魂原始格式仅作存档和调试用。

---

## 一、天凤格式（`data/games/*.json`）

这是项目的**核心数据格式**，由雀魂原始数据转换而来，结构参照天凤（tenhou.net）牌谱 JSON 设计。

### 1.1 顶层结构

```jsonc
{
  "ver": "2.3",                           // 格式版本，固定 "2.3"
  "ref": "260530-aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee",  // 牌谱 ID（来自雀魂 uuid）
  "log": [ /* 每一局 */ ],                // 核心：局数组
  "name": ["玩家A", "玩家B", "玩家C", "玩家D"],  // 4 家昵称，下标 = 座位号
  "dan": ["雀傑★2", "雀士★1", ...],       // 段位字符串
  "rate": [538, 189, 505, 0],             // 段位分
  "sx": ["F", "F", "F", "M"],             // 性别代码：F=女, M=男, C=其他
  "sc": [18000, -22, 30500, 20.5, ...],   // 最终点数 + 得点，共 8 个元素
  "rule": {
    "disp": "大会戦南喰赤",               // 规则显示名
    "aka51": 1,                           // 赤五万数量
    "aka52": 1,                           // 赤五饼数量
    "aka53": 1                            // 赤五索数量
  },
  "ratingc": "PF4",                       // PF4=四麻, PF3=三麻
  "lobby": 0,
  "title": ["大会戦南喰赤: 99999999", "1/1/2026, 12:00:00 AM"]
}
```

**`sc` 字段**：8 个元素，交替排列每位玩家的最终点数和得点（以千为单位）：

| 下标 | 含义 |
|------|------|
| `sc[0]` | 座位 0 最终点数（如 18000） |
| `sc[1]` | 座位 0 得点/1000（如 -22 表示 -22000 点） |
| `sc[2]` | 座位 1 最终点数 |
| `sc[3]` | 座位 1 得点/1000 |
| ... | ... |

**座位号**：0 = 起家东（第一局的庄家），1 = 起家南，2 = 起家西，3 = 起家北。

### 1.2 每局结构（`log[i]`）

每局是一个固定 17 元素的数组：

| 下标 | 含义 | 示例 |
|------|------|------|
| `[0]` | `[局码, 本场, 立直棒]` | `[0, 0, 0]` = 东一局 0 本场 0 立直棒 |
| `[1]` | `[4家初始点数]` | `[25000, 25000, 25000, 25000]` |
| `[2]` | `[宝牌指示牌...]` | `[45]` = 翻出一张宝牌指示牌：五饼 |
| `[3]` | `[里宝牌指示牌...]` | `[31]` = 里宝牌指示牌：一万（未立直时为空数组） |
| `[4]` | 座位 0 起手 13 张 | `[45, 52, 42, ...]` |
| `[5]` | 座位 0 摸牌序列 | `[16, 47, 12, "p272727", ...]` |
| `[6]` | 座位 0 弃牌序列 | `[42, 43, "r13", 60, ...]` |
| `[7]` | 座位 1 起手 13 张 | 同上 |
| `[8]` | 座位 1 摸牌序列 | 同上 |
| `[9]` | 座位 1 弃牌序列 | 同上 |
| `[10]` | 座位 2 起手 13 张 | 同上 |
| `[11]` | 座位 2 摸牌序列 | 同上 |
| `[12]` | 座位 2 弃牌序列 | 同上 |
| `[13]` | 座位 3 起手 13 张 | 同上 |
| `[14]` | 座位 3 摸牌序列 | 同上 |
| `[15]` | 座位 3 弃牌序列 | 同上 |
| `[16]` | 结局信息 | 见下方 |

#### 局码计算

局码 = `4 × 场风 + 局数`：

| 局码 | 含义 |
|------|------|
| 0 | 东一局 |
| 1 | 东二局 |
| 3 | 东四局 |
| 4 | 南一局 |
| 7 | 南四局 |
| 8 | 西一局 |

场风：0=东, 1=南, 2=西。

#### 结局信息（下标 `[16]`）

**和了**：

```jsonc
["和了",
  [-1300, 4700, -700, -700],      // 4 家点数变动
  [1, 1, 1,                        // [和牌者座位, 放铳者座位(自摸=和牌者), 和牌者座位, ...]
   "20符3飜700-1300点",            // 打点描述
   "門前清自摸和(1飜)",            // 役1
   "平和(1飜)",                    // 役2
   "立直(1飜)",                    // 役3
   "裏ドラ(0飜)"]                  // 役4（0飜 表示无里宝牌但仍列出）
]
```

多人和了（双响/三家和）时，后面会追加第二组和了信息。

**流局**：

```jsonc
["流局", [-1500, 1500, -1500, 1500]]   // 4 家点数变动（听牌罚符）
```

**流满**（流し満貫）：

```jsonc
["流し満貫", [4000, -2000, -2000, 0]]
```

**中途流局**（只有名称，无点数变动）：

```jsonc
["九種九牌"]
["四風連打"]
["四開槓"]
["三家和"]
["四家立直"]
```

### 1.3 牌面编码（天凤编码）

所有牌面用整数编码，规则为 `十位 = 花色，个位 = 数字`：

| 编码范围 | 花色 | 示例 |
|----------|------|------|
| 11–19 | 万子 (m) | 11=一万, 15=五万, 19=九万 |
| 21–29 | 饼子 (p) | 21=一饼, 25=五饼, 29=九饼 |
| 31–39 | 索子 (s) | 31=一索, 35=五索, 39=九索 |
| 41–47 | 字牌 (z) | 41=东, 42=南, 43=西, 44=北, 45=白, 46=发, 47=中 |
| **51** | **赤五万** | 红色的五万 |
| **52** | **赤五饼** | 红色的五饼 |
| **53** | **赤五索** | 红色的五索 |
| **60** | **摸切标记** | 仅出现在弃牌序列，表示摸什么打什么 |
| **0** | **大明杠占位** | 仅出现在弃牌序列，表示该位进行了大明杠 |

### 1.4 摸牌 / 弃牌序列中的特殊字符串

摸牌和弃牌序列的元素通常是整数（牌面编码），但鸣牌和立直操作会编码为字符串：

#### 立直标记：`r`

出现在**弃牌序列**中，`r` + 牌面编码：

| 字符串 | 含义 |
|--------|------|
| `"r25"` | 立直宣言，打出五饼 |
| `"r60"` | 立直宣言 + 摸切 |
| `"r13"` | 立直宣言，打出三万 |

#### 吃牌：`c`

出现在**摸牌序列**中，`c` + 三张牌的天凤编码（吃进的牌 + 手中两张）：

| 字符串 | 含义 |
|--------|------|
| `"c232425"` | 吃进三万(23)，手出四万(24)和五万(25) |
| `"c333234"` | 吃进三索(33)，手出二索(32)和四索(34) |

#### 碰牌：`p`

出现在**摸牌序列**中，三张相同牌 + `p` 前缀标记来源方向：

| `p` 的位置 | 来源方向 |
|-----------|----------|
| 第 0 位 `"p272727"` | 上家 |
| 第 1 位 `"27p2727"` | 对家 |
| 第 2 位 `"2727p27"` | 下家 |

#### 大明杠：`m`

出现在**摸牌序列**中，格式同碰但为四张牌：

| 字符串 | 含义 |
|--------|------|
| `"m39393939"` | 从上家大明杠九索 |
| `"39m393939"` | 从对家大明杠九索 |

同时弃牌序列会写入一个 `0` 作为占位。

#### 加杠（小明杠）：`k`

出现在**弃牌序列**中，替换原有碰串的 `p` 为 `k` + 加杠的那张牌：

| 字符串 | 含义 |
|--------|------|
| `"k37373737"` | 加杠七索（原来碰的七索） |

#### 暗杠：`a`

出现在**弃牌序列**中，四张牌中第 3 和第 4 张之间插入 `a`：

| 字符串 | 含义 |
|--------|------|
| `"121212a12"` | 暗杠二万 |

#### 拔北（三麻专用）：`f`

出现在**弃牌序列**中：

| 字符串 | 含义 |
|--------|------|
| `"f44"` | 拔了一张北风(44) |

### 1.5 完整示例：解读一局

```jsonc
// 东一局，0 本场，0 立直棒
[0, 0, 0],
// 4 家初始点数
[25000, 25000, 25000, 25000],
// 宝牌指示牌：五饼(25 → 宝牌是六饼)
[45],
// 里宝牌指示牌：一万
[31],
// 座位 0 起手 13 张
[45, 52, 42, 25, 38, 43, 18, 34, 17, 37, 28, 44, 31],
// 座位 0 摸牌序列
[16, 47, 12, 31, 32, 15, 33, 34, 32, 22, 29, 11, 23, 34, 45, 12, 21],
// 座位 0 弃牌序列
[42, 43, 44, 45, 47, 28, 31, 12, 31, 60, 60, 60, 60, 32, 60, 60, 60],
// ... 座位 1/2/3 的起手/摸/弃 ...
// 结局：
["和了",
  [-1300, 4700, -700, -700],
  [1, 1, 1, "20符3飜700-1300点",
   "門前清自摸和(1飜)", "平和(1飜)", "立直(1飜)", "裏ドラ(0飜)"]]
```

解读：座位 1 自摸和了，20 符 3 飜，闲家各付 700 点、庄家付 1300 点。役种为门前清自摸和 + 平和 + 立直，无里宝牌。

## 二、雀魂原始格式（`data/raw/*.json`）

这是雀魂 protobuf 协议解码后的直接产物，保留所有原始细节。文件较大（约 400KB+），包含大量 UI 交互信息。

### 2.1 顶层结构

```jsonc
{
  "head": { /* 牌谱元数据 */ },
  "actions": [ /* 按时间顺序的事件流 */ ]
}
```

### 2.2 `head`：牌谱元数据

```jsonc
{
  "uuid": "260530-aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee",  // 牌谱唯一 ID
  "start_time": 1700000000,              // 开始时间（Unix 时间戳）
  "end_time": 1700002627,                // 结束时间

  "accounts": [                          // 玩家信息（4 位）
    {
      "account_id": 10000001,
      "nickname": "示例玩家1",
      "seat": 1,                         // 座位号
      "avatar_id": 40012503,
      "character": {                     // 角色装扮信息
        "charid": 20000125,
        "level": 5,
        "skin": 40012503,
        "is_upgraded": true,
        "extra_emoji": [928]
      },
      "level": { "id": 10103, "score": 23 },    // 四麻段位
      "level3": { "id": 20102 },                // 三麻段位
      "title": 600009,                          // 称号 ID
      "views": [ /* 装饰物品 */ ]
    }
    // ... 其余 3 位玩家
  ],

  "config": {
    "category": 4,                       // 游戏分类
    "mode": {
      "mode": 2,                         // 1=东风战, 2=半庄战（南）
      "detail_rule": {
        "dora_count": 3,                 // 赤 5 总数
        "init_point": 25000,             // 初始点数
        "fandian": 30000,                // 返点
        "shiduan": 1,                    // 食断（1=有喰断）
        "liqibang_value": 1000,          // 立直棒点数
        "changbang_value": 300,          // 本场棒点数
        "have_biao_dora": true,          // 表宝牌
        "have_li_dora": true,            // 里宝牌
        "have_yifa": true,               // 一发
        "have_nanruxiru": true,          // 南入/西入
        "have_liujumanguan": true,       // 流满
        "time_fixed": 5,                 // 思考时间（秒）
        "time_add": 20,                  // 追加时间（秒）
        "noting_fafu_1": 1000,           // 不听罚符（1人）
        "noting_fafu_2": 1500,           // 不听罚符（2人）
        "noting_fafu_3": 3000,           // 不听罚符（3人）
        "shunweima_2": 5,               // 二位马点
        "shunweima_3": -5,              // 三位马点
        "shunweima_4": -15,             // 四位马点
        "fanfu": 1                       // 翻缚（最低一番）
      }
    },
    "meta": {
      "contest_uid": 99999999            // 大会战 ID（如有）
    }
  },

  "result": {
    "players": [
      { "seat": 1, "total_point": 58300, "part_point_1": 68300 },
      { "seat": 2, "total_point": -3700, "part_point_1": 16300 },
      { "seat": 3, "total_point": -20600, "part_point_1": 9400 },
      { "total_point": -34000, "part_point_1": 6000 }  // seat 0（缺省）
    ]
  },
  "standard_rule": 1
}
```

**`result` 字段说明**：
- `total_point`：最终得点（相对返点的差值 × 1000，如 58300 表示 +58.3 个点）
- `part_point_1`：最终持点（实际剩余点数）

### 2.3 雀魂牌面表示

雀魂使用**字符串**表示单张牌：

| 格式 | 含义 | 示例 |
|------|------|------|
| `Nm` | N 万 | `"3m"` = 三万, `"9m"` = 九万 |
| `Np` | N 饼 | `"5p"` = 五饼, `"1p"` = 一饼 |
| `Ns` | N 索 | `"7s"` = 七索, `"0s"` = 赤五索 |
| `Nz` | 字牌 | `"1z"` = 东, `"7z"` = 中 |
| `0m`/`0p`/`0s` | 赤 5 | `"0m"` = 赤五万, `"0p"` = 赤五饼 |

> 注意：`"0s"` 等以 `0` 开头的表示赤 5，不是零索。

### 2.4 `actions`：事件流

`actions` 是按时间顺序排列的事件数组，每个元素有 `name`（事件类型）和 `data`（事件数据）。

#### `RecordNewRound` — 新一局开始

```jsonc
{
  "name": "RecordNewRound",
  "data": {
    "scores": [25000, 25000, 25000, 25000],    // 4 家初始点数
    "tiles0": ["2p", "8p", "1z", ...],         // 座位 0 起手 13 张
    "tiles1": ["7s", "8p", "6z", ...],         // 座位 1 起手 13 张
    "tiles2": ["2m", "9p", "2z", ...],         // 座位 2 起手 13 张
    "tiles3": ["1s", "5z", "8m", ...],         // 座位 3 起手 13 张
    "doras": ["1m"],                            // 初始宝牌指示牌
    "left_tile_count": 69,                      // 剩余牌数
    "paishan": "2p8p1z9m7s8p...",               // 牌山完整排列（用于验证）
    "opens": [ /* 各家副露（开局时为空） */ ]
  }
}
```

> 注意：庄家（当局的东家）起手有 14 张牌（多出的第 14 张是庄家的第一张摸牌）。

#### `RecordDealTile` — 摸牌

```jsonc
{
  "name": "RecordDealTile",
  "data": {
    "seat": 1,                    // 摸牌者座位（省略 = 座位 0）
    "tile": "7z",                 // 摸到的牌
    "left_tile_count": 68,        // 剩余牌数
    "doras": [],                  // 新增宝牌指示牌（杠后翻宝牌时非空）
    "operation": { /* 可执行操作提示（UI 用） */ }
  }
}
```

#### `RecordDiscardTile` — 弃牌

```jsonc
{
  "name": "RecordDiscardTile",
  "data": {
    "seat": 1,                    // 弃牌者座位（省略 = 座位 0）
    "tile": "9s",                 // 打出的牌
    "moqie": false,               // 是否摸切（true = 摸什么打什么）
    "is_liqi": false,             // 是否立直宣言
    "zhenting": [false, false, false, false],  // 4 家振听状态
    "tingpais": [                 // 该玩家当前听牌信息（可选）
      {
        "tile": "6p",
        "haveyi": true,           // 有役
        "count": 1,               // 荣和番数
        "fu": 40,                 // 荣和符数
        "count_zimo": 1,          // 自摸番数
        "fu_zimo": 40             // 自摸符数
      }
    ],
    "doras": [],                  // 新增宝牌指示牌
    "operations": [ /* 其他玩家的可执行操作提示 */ ]
  }
}
```

#### `RecordChiPengGang` — 吃 / 碰 / 大明杠

```jsonc
{
  "name": "RecordChiPengGang",
  "data": {
    "seat": 3,                    // 鸣牌者座位
    "type": 1,                    // 0=吃, 1=碰, 2=大明杠
    "tiles": ["5z", "5z", "5z"], // 组成面子（或杠子）的牌
    "froms": [3, 3, 2],           // 每张牌的来源座位
    "tile_states": [0, 0],
    "scores": [],                 // 点数变动（如有）
    "zhenting": [false, false, false, false]
  }
}
```

#### `RecordAnGangAddGang` — 暗杠 / 加杠

```jsonc
{
  "name": "RecordAnGangAddGang",
  "data": {
    "seat": 2,                    // 杠牌者座位
    "type": 3,                    // 3=暗杠, 2=加杠（小明杠）
    "tiles": "5m",                // 暗杠/加杠的牌
    "doras": ["8m"],              // 杠后翻出的新宝牌指示牌
    "zhenting": [false, false, false, false]
  }
}
```

#### `RecordBaBei` — 拔北（三麻专用）

```jsonc
{
  "name": "RecordBaBei",
  "data": {
    "seat": 0,
    "moqie": false
  }
}
```

#### `RecordHule` — 和了

```jsonc
{
  "name": "RecordHule",
  "data": {
    "hules": [                    // 和了信息（双响/三家和时有多条）
      {
        "seat": 2,                // 和牌者座位
        "hu_tile": "3s",          // 和的牌
        "hand": ["2p", "3p", "4p", "3s"],          // 手牌
        "ming": ["kezi(2z,2z,2z)", "shunzi(4m,6m,5m)"],  // 副露
        "doras": ["8m"],          // 宝牌
        "li_doras": [],           // 里宝牌
        "fans": [                 // 役列表
          { "id": 11, "val": 1 } // id 对应雀魂内部役 ID，val = 番数
        ],
        "count": 1,               // 总番数
        "fu": 30,                 // 符数
        "point_rong": 1000,       // 荣和点数
        "point_zimo_qin": 0,      // 自摸时庄家支付（荣和时为 0）
        "point_zimo_xian": 0,     // 自摸时闲家支付（荣和时为 0）
        "point_sum": 8600,        // 总得点（含本场等加成）
        "dadian": 1000,           // 打点
        "zimo": false,            // 是否自摸
        "qinjia": false,          // 是否庄家
        "liqi": false             // 是否立直
      }
    ],
    "old_scores": [7000, 68300, 15300, 9400],     // 和了前 4 家点数
    "delta_scores": [-1000, 0, 1000, 0],           // 4 家点数变动
    "scores": [6000, 68300, 16300, 9400],          // 和了后 4 家点数
    "doras": [],
    "gameend": { "scores": [] }                    // 若终局则包含最终结算
  }
}
```

#### `RecordNoTile` — 流局

```jsonc
{
  "name": "RecordNoTile",
  "data": {
    "scores": [ /* 各家听牌/不听信息 */ ],
    "delta_scores": [-1500, 1500, -1500, 1500],   // 不听罚符
    "liujumanguan": false                          // 是否流满
  }
}
```

#### `RecordLiuJu` — 中途流局

```jsonc
{
  "name": "RecordLiuJu",
  "data": {
    "type": 1     // 1=九种九牌, 2=四风连打, 其他值由程序推断
  }
}
```

## 三、两种格式的对应关系

| 维度 | 雀魂原始格式 (`raw/`) | 天凤格式 (`games/`) |
|------|----------------------|---------------------|
| 牌面表示 | 字符串：`"3m"`, `"5p"`, `"0s"` | 整数编码：13, 25, 53 |
| 事件表达 | 扁平事件流 (`actions[]`) | 按局 + 按座位分组 (`log[][]`) |
| 鸣牌 | `RecordChiPengGang` + `type` 字段 | 字符串前缀 `c`/`p`/`m`/`k`/`a` |
| 立直 | `is_liqi: true` 字段 | `"r"` 前缀 |
| 摸切 | `moqie: true` 字段 | 数字 `60` |
| 和了 | `RecordHule` + 详细 `hules` | `["和了", delta, detail]` |
| 流局 | `RecordNoTile` | `["流局", delta]` |
| 中途流局 | `RecordLiuJu` | `["九種九牌"]` 等 |
| 玩家信息 | `head.accounts` (详细) | `name`/`dan`/`rate`/`sx` (精简) |
| 规则 | `head.config.mode.detail_rule` | `rule.disp` + `aka5x` |
| 文件大小 | ~400KB | ~8-13KB |

### 牌面编码转换公式

雀魂字符串 → 天凤整数：

```
tm2t("Nm") = 10 + N     // 万子：11-19
tm2t("Np") = 20 + N     // 饼子：21-29
tm2t("Ns") = 30 + N     // 索子：31-39
tm2t("Nz") = 40 + N     // 字牌：41-47
tm2t("0m") = 51          // 赤五万
tm2t("0p") = 52          // 赤五饼
tm2t("0s") = 53          // 赤五索
```

## 四、文件名规则

两种格式的文件名相同，格式为：

```
{日期}-{UUID}_{玩家ID}.json
```

示例：`260530-aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee_a10000001.json`

- `260530`：日期（2026 年 5 月 30 日，即 26 年 05 月 30 日）
- `aaaaaaaa-bbbb-cccc-dddd-eeeeeeeeeeee`：牌谱 UUID
- `a10000001`：目标玩家的 account_id

同一 UUID 的 `raw/` 和 `games/` 文件是同一场对局的两种表示。

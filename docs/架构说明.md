# 架构说明

> 给"以后要修改这个项目的人"看。基础用法见根 `README.md`。

## 一、设计原则

**功能解耦**：每个模块单一职责，彼此通过纯数据传递。任何一块换实现（比如 Playwright 换成 chrome-devtools-mcp）只动一个文件。

**只做必要的事**：抓取产出**天凤格式 json**就停。统计、AI 复盘是另一类工具的活（`stats.py` 是顺手的简单实现；要 AI 复盘喂给 mjai-reviewer / NAGA 即可）。

**第三方代码不修改**：`vendored/kbkn3/` 逐字保留，要修改改 `inject/src/entry.ts` 这一层包装。Apache-2.0 要求修改要标注，分离开省麻烦。

## 二、模块职责

```
fetch.py  ──→  src/fetcher/cli.py  ──→ browser.py  →  capture.py  →  tenhou_export.py
                                          │              │                   │
                                       Playwright    截 WS 帧 +          dict 写文件
                                       启动 +        调 inject.js
                                       注入

stats.py  ──→  src/parser/tenhou.py  ──→  tiles.py
              json → 局对象          牌面编解码
```

| 模块 | 输入 | 输出 | 不关心 |
|---|---|---|---|
| `src/fetcher/browser.py` | profile 路径、headless 开关 | 一个 `BrowserContext` | 雀魂、协议、文件 |
| `src/fetcher/capture.py` | `BrowserContext` + 牌谱 URL | 天凤 dict | 文件、CLI 参数 |
| `src/fetcher/tenhou_export.py` | dict + 输出路径 | 写文件、返回路径 | 浏览器、网络 |
| `src/fetcher/cli.py` | 命令行参数 | exit code | 业务逻辑 |
| `inject/src/entry.ts` | 二进制帧 (b64) | 解码后对象 | 怎么被调用、怎么落盘 |
| `src/parser/tenhou.py` | 天凤 json | 结构化局对象 | 抓取、统计 |
| `stats.py` | 多个局对象 | 终端报告 | 解析、抓取 |

## 三、关键技术点

### 3.1 抓 WebSocket 的方式

走 Playwright 的 `page.on("websocket")` 监听 + `framesent`/`framereceived` 事件。**不解 Wrapper 层**——通过裸搜请求名字符串 `b".lq.Lobby.fetchGameRecord"` 匹配 sent 帧，记下 index；recv 时按 index 配对。这样避开了在 Python 侧再装一份 protobuf。

雀魂帧首 3 字节：`type(1) + index(2 LE)`。type=`0x02` 是 request，`0x03` 是 response。

### 3.2 解码链（kbkn3 内核）

```
binary frame
  ↓ liqi-frame.ts: 解外层 Wrapper（双重包装）
  ↓ record-decode.ts: 处理 records[] / actions[].result 两种结构
  ↓ kyoku.ts: 事件 handler，归一化为局对象
  ↓ dd.ts (parse): 出天凤 json
天凤 json
```

每一步都是纯函数，能在 Playwright 的 `page.evaluate` 里跑（解码完拿对象回 Python 写盘）。

注入时机：`page.add_init_script()` 在 document_start 跑，早于雀魂 Unity WebGL 启动，保证 hook 能挂到 `window.WebSocket`。

### 3.3 protobufjs 必须 v8

kbkn3 的 `liqi-proto.js` 是 pbjs 静态生成（避免 protobuf 反射触发 MV3 CSP `unsafe-eval`）。这版生成代码用了 `reader.tag()`，**v7 没这个 API**。`inject/package.json` 锁了 `protobufjs ^8.4.0`。

### 3.4 esbuild 解析 vendored 包的依赖

`vendored/kbkn3/assets/majsoul/liqi-proto.js` 引 `protobufjs/minimal.js`。esbuild 默认从模块所在目录向上找 `node_modules`，会一路上溯到项目根都找不到（项目根没装 npm）。解法：`vendored/kbkn3/node_modules` **软链** 到 `inject/node_modules`。

软链已入库（git 支持 symlink）。clone 下来之后 `cd inject && npm install` 重建 inject 的 node_modules，软链就指向有效目标。

### 3.5 登录态隔离

`data/profile/` 是 Playwright 的持久化用户数据目录，跟你日常 Chrome 完全隔离。首次 `--login-only` 扫码进大厅后保存，之后每次 launch 复用。

profile 大小约 250~310MB（雀魂 Unity 资源缓存进来了），**绝不入库**：`.gitignore` + `chmod 700`（`browser.py` 自动设权限）。

## 四、协议变更怎么办

雀魂偶尔会改 liqi 协议（约一年一次）。表现：`fetch.py` 跑出来 protobuf 解码报错，或字段缺失。

**正确做法是跟随 kbkn3 上游**：

```bash
# 看 kbkn3 是否发了新 release
curl -s https://api.github.com/repos/kbkn3/MahjongSoul-review-supporter/releases/latest \
  | jq -r .tag_name

# 拉新 tarball，更新 vendored
TAG=v1.6.0  # 例
curl -L https://github.com/kbkn3/MahjongSoul-review-supporter/archive/refs/tags/$TAG.tar.gz \
  -o /tmp/kbkn3.tar.gz
tar xzf /tmp/kbkn3.tar.gz -C /tmp/
cp /tmp/MahjongSoul-review-supporter-*/src/lib/*.ts vendored/kbkn3/lib/
cp /tmp/MahjongSoul-review-supporter-*/src/content-scripts/dd.ts vendored/kbkn3/content-scripts/
cp /tmp/MahjongSoul-review-supporter-*/src/assets/majsoul/liqi-proto.js vendored/kbkn3/assets/majsoul/
cp /tmp/MahjongSoul-review-supporter-*/src/assets/majsoul/liqi-proto.d.ts vendored/kbkn3/assets/majsoul/
cp /tmp/MahjongSoul-review-supporter-*/src/assets/majsoul/cfg/*.json vendored/kbkn3/assets/majsoul/cfg/

# 改 vendored/kbkn3/NOTICE.md 把 commit/tag 更新

# 重 build
cd inject && npm run build && cd -

# 重测
python3 stats.py data/games/<已有>.json   # 验下游没坏
python3 fetch.py "<已有牌谱链接>" --force   # 验上游
```

## 五、扩展点

可能想做但没做的：

- **统计指标更全**：当前 `stats.py` 是 100 行的简单实现，没覆盖立直收支、副露分级、安定段位等。继续扩按 `参考资料/历史文档/最初方案.md` 的指标体系来。
- **HTML 报告**：终端文本不直观，可以加个 `report.py` 出 HTML / 对标基准线对比。
- **AI 复盘**：拿天凤 json 直接喂 mjai-reviewer 即可（已克隆在 `参考资料/三方代码/mjai-reviewer/`）。
- **批量管理**：从 amae-koromo 等站抓"该玩家最近 N 局 paipu UUID 列表"，再喂 fetch.py 全部拉下来。⚠ 警惕封号风险，加大延迟。

每一项都是独立增量，不影响现有抓取/解析链路。

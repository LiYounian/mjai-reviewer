# 雀魂 token / 牌谱获取 — 2026-06-01 实测结论

> 本次用新装的 `chrome-devtools-mcp` 插件全程自动操控浏览器，尝试解决"提取登录 token → 拉牌谱"这个卡点。
> **核心结论：token 提取成功，但雀魂新版客户端已把 token 这条路彻底封死，提取到的 token 不可用。**

## 一、做成了什么

1. **成功用 chrome-devtools-mcp 自动读出 localStorage**（全程脚本操控，无需手动粘代码）：
   - 这版雀魂（**Unity 2022.3 WebGL，客户端 4.0.43**）把一串 UUID 明文存在 `localStorage.access_token`。
   - 同时 localStorage 里还明文存了 `account`(邮箱) 和 `password`(明文密码) —— 雀魂自己存的"记住密码"，注意安全。
   - 已存进 `data/config.json`（chmod 600，仅 token + account_id + 邮箱，**不存密码**）。
2. **tensoul 跑通到"连上国服服务器并发起登录"**，并为国服打了兼容补丁（见下）。

## 二、撞上的两堵墙（token 路线被证伪）

### 墙 1：localStorage 里的 UUID 不是能登录的 token
- 用 tensoul 发 `oauth2Login {type:10, access_token:<那个UUID>}`，返回 **error code 109**（不是封号，封号是 151）。
- 调研结论：那个 UUID 形态/作用更像 `random_key`（设备标识），**不是 oauth2 能用的 access_token**。
- 雀魂真正能 `oauth2Login` 的 token 必须来自：① passport 响应（且必须配对 `uid`）；或 ② 账密 `ReqLogin` 且带 `gen_access_token=True`。
- 且**国服的登录 `type` 不是 tensoul 默认的 10**（10 是海外 Yo-star 服的值），国服走 maj-soul 自己的通行证体系，type 值需另抓。

### 墙 2：网页内直接调 fetchGameRecord 也不通
- 参考项目 MajsoulPaipuAnalyzer 的老办法是在已登录网页里调 `app.NetAgent.sendReq2Lobby(...)` 拉牌谱，绕过登录。
- 但**新版雀魂是纯 Unity WebGL**（`unity-canvas` / `createUnityInstance` / `unityFramework`），游戏逻辑全编译进 wasm/Unity 闭包，JS 层 `app`/`GameMgr`/`NetAgent` **全部 undefined，读不到**。
- 参考项目那套是老版 **Cocos 引擎**时代的实现，对新版已失效。

**一句话**：之前"卡在提取 token"，更深层的真相是——**就算提取到也用不了**。这不是工具问题，是雀魂新版客户端架构升级（Cocos→Unity WebGL）把所有非官方提取手段都封死了。

## 三、对 tensoul 的国服兼容补丁（已落盘）

`tensoul参考/server_config.js` 第 36 行附近：国服 `config.json` 用 `ip[0].gateways[].url`，海外服用 `ip[0].region_urls[].url`，已改成两者兼容 + 缺失返回空数组（避免崩溃）。

国服正确参数（已验证能连上、能发登录请求，只是 token 被拒）：
- `MJS_BASE="https://game.maj-soul.com/1"`（**必须带 `/1`**，根路径 404）
- `MJS_GATEWAY="wss://route-5.maj-soul.com/gateway"`（直接指定网关，跳过 discovery；网关列表来自 `v<version>/config.json` 的 `ip[0].gateways`）
- `ACCESS_TOKEN`、用法 `node . <牌谱ID>`

## 四、下一步的候选路线（牌谱获取另谋，待下次决定）

调研给出的、理论可行的方案，按推荐度：

1. **现成工具导出（README 首选策略）**：用 amae-koromo（雀魂牌谱统计站）/ 油猴脚本 / mjai-reviewer 等成熟轮子登录导出天凤 json，本项目只负责解析+统计。**最省事、风险最低。**
2. **账密登录路线**：用 `MahjongRepository/mahjong_soul_api`（国服支持最好），`ReqLogin` 用 账号 + `hmac_sha256(key="lailai", msg=密码)` + `gen_access_token=True` 拿到**真正可用的 token**，再喂 tensoul。
   - ⚠️ 需新搭 Python 协议栈；且是**账密直连雀魂**，封号风险比抓 token 更高，需用户明确承担。
3. 抓一帧国服真实 `oauth2Login` WebSocket 帧，解出国服实际 `type` 值——但 token 来源问题不解决，光有 type 也没用。

只要用上面任一方式拿到**天凤 json**，直接 `python3 src/stats.py <file.json>` 即可出统计。管线已跑通，不依赖 token 问题。
